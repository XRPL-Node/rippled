enum class xrpl::test::Invariants_test::TxAccount : int
strongprivate

Run a specific test case to put the ledger into a state that will be detected by an invariant.

Simulates the actions of a transaction that would violate an invariant.

Parameters
expect_logsOne or more messages related to the failing invariant that should be in the log output @precheck See "Precheck" above @fee If provided, the fee amount paid by the simulated transaction. @tx A mock transaction that took the actions to trigger the invariant. In most cases, only the type matters. @ters The TER results expected on the two passes of the invariant checker. @preclose See "Preclose" above. Note that @preclose runs before @precheck, but is the last parameter for historical reasons @setTxAccount optionally set to add sfAccount to tx (either A1 or A2)
